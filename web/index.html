What it does
Everything you need to manage your rail network

Built for large factories — scales from your first train to a 1000-station megabase without any manual setup.

📡
Zero-config auto-discovery
Every train stop is tracked the moment it's built, removed, renamed, cloned, or revived via blueprint. No setup. No station IDs. No circuit wiring required.
📊
Grouped resource view
Stations are automatically grouped by resource (item or fluid). See load vs unload counts, trains present, queue saturation, and average wait time — all in one row.
🗺️
Congestion heatmap
Opt-in map overlay that draws a colour-coded circle at every tracked station. Cool (green) = trains flowing. Hot (red) = starved. Spot problems at a glance.
📉
Throughput graphs
Expand any resource to see a 10-minute trains-served-per-minute history, averaged and peak values — no external tools needed.
⚠️
Supply alerts
Native Factorio alerts fire when a resource has consumers but no available provider — same alert panel as low-power warnings, with a count badge in the dashboard.
⚡
Per-station controls
Zoom to station, open the assigned train, or enable/disable a stop directly from the dashboard. Disabling sets the train limit to 0 — no new trains dispatched.

🏷️ Station naming convention

The mod discovers stations whose names contain a rich-text icon tag followed by Load or Unload. Depots, junctions, and any other stops without this pattern are silently ignored.

[img=item/iron-plate] Load ← load station [img=item/iron-plate] Unload ← unload station [img=fluid/crude-oil] Load ← fluids work too [img=item/copper-plate] Unload #2 ← suffixes like "#2" are fine
Under the hood
Clean architecture

Each module has exactly one responsibility. events.lua is the only file that calls script.on_event — everything else exposes plain functions.

Module Responsibility
control.luaEntry point — loads modules, calls events.register(). Nothing else.
scripts/parser.luaPure, stateless name parsing → {kind, proto, mode, group_key}
scripts/cache.luaOwns the entire storage schema. All state access goes through here.
scripts/discovery.luaKeeps cache in sync via events; one full scan on init only.
scripts/statistics.luaQueue detection, saturation, enable/disable, average-wait tracking.
scripts/throughput.luaTrains-served-per-minute ring buffer (10-min window, per group).
scripts/heatmap.luaCongestion map overlay — per-player, opt-in, rendering API.
scripts/alerts.lua"No available provider" alerts — native Factorio + dashboard badge.
scripts/rendering.luaSort / filter logic + row widget construction (view only, no state).
scripts/gui.luaOwns the button + window; builds, opens, refreshes, routes clicks.
scripts/events.luaSingle event-wiring hub. Defines the cadence (1s stats, 1min throughput roll).
